INTRODUCCIÓN

La API (interfaz de programación de aplicaciones) de TalkFood, actualmente le permite acceder a un modelo dirigido a la administracion de una receta, mediante la cual puede administrar los registros en la base de datos de recetas con datos falsos, especificamente para prueba.
Puede utilizar las API para crear sus propios sitios web o aplicaciones dinámicos respaldados por los mismos datos que se generan como datos falsos atraves de factories y datos semillas en laravel.
Con la API para Recipe actualmente puede:
Buscar mediante el id y muestre la información de una receta específica. Además consultar listado completo de recetas. También le permite agregar sus recetas , eliminarlas y actualizarlas.
Esta es una API RESTful, lo que significa que utiliza URL predecibles para acceder a los recursos y, en caso de error, devuelve códigos de respuesta HTTP significativos. Como es el caso del error 404, al consultar el id de una receta no existente. Esto permite el uso de autenticación GET, POST y HTTP, que los clientes HTTP estándar entienden.
